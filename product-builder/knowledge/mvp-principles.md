# MVP Principles — Product Builder

O que é (e o que não é) um MVP real.

---

## O MVP Real

Um MVP (Minimum Viable Product) é o menor produto possível que:
1. Resolve o problema real do usuário
2. Entrega valor suficiente para alguém pagar ou usar
3. Valida hipóteses do negócio
4. Permite aprender o que construir a seguir

**Não é:** versão com todos os bugs, sem design, sem UX

---

## Teste do MVP

Para cada feature que está "no MVP", perguntar:
> "Se lançar sem esta feature, o produto ainda resolve o problema central?"

- Se SIM → fora do MVP
- Se NÃO → no MVP

---

## Sinais de MVP Inflado

- Mais de 8 features "essenciais"
- Features que dependem umas das outras em cadeia de 5+
- "Precisamos de analytics", "precisa ter dark mode", "e o app mobile?"
- Funcionalidades que apenas ~5% dos usuários usariam no primeiro mês

---

## Sinais de MVP Muito Simples

- Não resolve o problema de ponta a ponta
- Usuário não consegue completar a jornada principal
- Missing alguma validação de negócio crítica
- Sem feedback ao usuário após ação

---

## Princípios de Escopo do MVP

### 1. Jornada Completa > Features Completas
Melhor ter 1 fluxo completo do que 5 fluxos pela metade.

### 2. Menos = Mais Claro
Menos features = mais fácil de testar, corrigir e aprender.

### 3. Escopo Explícito
"Fora do MVP" é tão importante quanto "no MVP".  
Liste o que ficou de fora para não criar expectativas.

### 4. Validação Primeiro
O MVP deve provar que o problema existe e que a solução funciona — antes de construir mais.

---

## Exemplos de MVP Bem Escoped

### Aplicativo de Tarefas
❌ MVP Inflado: criar, editar, excluir, categorias, tags, subtarefas, compartilhar, notificações, relatórios, app mobile, dark mode

✅ MVP Real: criar tarefa (título + data), listar tarefas, marcar como concluída, excluir

### E-commerce
❌ MVP Inflado: catálogo completo, filtros, favoritos, reviews, cupons, frete calculado, área do vendedor, relatórios

✅ MVP Real: listar produtos, carrinho simples, checkout com Stripe, email de confirmação
